What Happens When You Use the Dashboard

This app is a private logbook for time, money, and calories. You add entries, then the dashboard turns them into totals, charts, recent history, and trends.

Start with the click you already understand

Imagine you sign in, choose the calories tab, search for a food, and press Add Entry. That one action travels through a frontend, an API, validation, authentication, and a database.

UI

You see pages

Dashboard, entries, and analytics live under the protected app shell.

API

The browser asks

Fetch calls send create, edit, delete, and search requests.

DB

Postgres remembers

Prisma writes each entry with the signed-in user's ID.

The shell wraps every protected page

Think of the app shell like a reusable dashboard frame: side navigation, top bar, metric color theme, and shared entry data all stay in place while the page content changes.

This layout receives whatever page is being viewed.

Suspense lets the page wait safely for browser-only state.

EntriesProvider makes entry data available everywhere inside the shell.

DashboardLayout supplies the shared sidebar, topbar, and page frame.
