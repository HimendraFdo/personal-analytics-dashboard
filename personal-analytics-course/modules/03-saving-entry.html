The browser builds the request

A hook is a tidy place for repeated browser behavior. Here it turns form data into the exact shape the server expects.

CODE 
const input: EntryInput = {
  title: payload.title,
  value: payload.value,
  metricType: activeMetric,
  category: payload.category ?? DEFAULT_ENTRY_CATEGORIES[activeMetric],
  date: payload.date.toISOString(),
  note: payload.note,
};
const created = await createEntry(input);
setEntries((current) => [created, ...current]);
PLAIN ENGLISH

Create the object that will travel to the backend.

Attach the active metric tab, such as time, money, or calories.

Use a default category if the form did not provide one.

Convert the date into a safe string for the API.

Send it, wait for the saved copy, then put it at the top of the visible list.

The server checks the badge before the database

Server routes are the security desk. They verify the signed-in user and reject bad shapes before Prisma is allowed to write anything.

CODE 
const { userId } = await auth();

if (!userId) {
